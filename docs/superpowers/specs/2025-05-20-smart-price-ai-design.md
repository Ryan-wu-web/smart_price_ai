## 1. 产品定位与核心体验

### 1.1 产品定位

面向 **C 端消费者**的 AI 智能购物助手，核心解决"看到想买但难以检索"的痛点。

区别于传统电商 App 的"人找货"，打造 **"货找人 + AI 辅助决策"** 的全新购物体验。

### 1.2 核心用户旅程（Happy Path）

```
用户打开 App → 拍摄商品照片 → AI 识别出"耐克 Air Max 黑色 42码"
    ↓
展示识别结果 + 动态生成建议卡片（查看同款低价 / 只看旗舰店 / 相似推荐 / 价格趋势）
    ↓
用户点击"查看同款低价" → 展示跨平台比价列表（淘宝¥899 / 京东¥849 / 拼多多¥799）
    ↓
用户输入"1000元以内，黑色，评价4.8以上" → NL 筛选实时刷新结果
    ↓
点击商品 → 查看详情 + AI 价格趋势分析（"历史均价¥850，当前价低于均价12%，建议购买"）
    ↓
一键生成决策报告卡片 → 保存 / 分享
```

### 2.3 模块化设计原则

- **识物模块**、**建议模块**、**比价模块**、**筛选模块**、**趋势模块**、**多目标检测模块**、**报告生成模块**、**AI 导购模块** — 八模块独立，通过统一接口交互
- **数据源抽象层 (DataSource)** — 比价数据统一走此层，Mock 数据源和真实 API 数据源实现同一接口，随时切换
- **LLM 编排引擎** — 集中管理所有 Prompt 模板、结构化输出解析、错误重试逻辑，AI 模型可插拔替换

## 3. 核心功能模块详细设计

### 3.1 拍照识物模块 (Recognition)

**输入**：用户上传的商品图片（Base64 或文件流）
**输出**：结构化识别结果 `{category, brand, color, style, attributes, confidence}`

**流程**：
1. App 拍摄/选择图片 → 压缩后上传至 `/api/v1/recognize`
2. 服务端将图片转 Base64 → 调用火山引擎 Doubao-VLM（多模态模型）
3. VLM 返回商品描述文本
4. **LLM 二次解析**：将描述文本结构化提取为 `{category, brand, color, ...}`
5. 返回给 App 展示

**关键设计**：VLM + LLM 两阶段编排（体现"复杂 AI 任务编排"评审加分点）

### 3.2 智能建议卡片模块 (Suggestion)

**输入**：识别结果 `{category, brand, color}`
**输出**：动态生成的建议卡片列表 `[{type, title, icon, action}]`

**卡片类型**：

| 类型 | 示例 | 触发条件 |
|------|------|---------|
| `lowest_price` | "查看同款低价" | always |
| `official_store` | "只看官方旗舰店" | brand 不为空时 |
| `similar_style` | "相似风格推荐" | always |
| `price_trend` | "查看价格走势" | always |
| `filter_color` | "筛选：深蓝色" | color 不为空时 |

**实现**：LLM 根据识别结果动态决定生成哪些卡片（体现 Prompt 工程能力）

### 3.3 跨平台比价模块 (Comparison)

**输入**：识别关键词 `{category, brand, color, style}`
**输出**：聚合后的商品列表 `[{id, name, price, platform, url, tags, rating}]`

**数据源抽象层设计**：

```python
class DataSource(ABC):
    @abstractmethod
    async def search(self, query: SearchQuery) -> list[Product]: ...

class MockDataSource(DataSource): ...      # 当前使用
class TaobaoDataSource(DataSource): ...    # 预留接入
class JDDataSource(DataSource): ...        # 预留接入
```

**Mock 数据策略**：预设 200-300 条跨平台商品数据，覆盖 4 大品类（鞋服/数码/美妆/家居），每个商品 3 个平台价格。

**未知商品 Fallback**：当识别结果无精确匹配时，LLM 动态生成参考价（标注"AI 参考价"）+ 推荐相似款。

### 3.4 自然语言筛选模块 (Filter)

**输入**：用户自然语言文本 + 当前商品列表
**输出**：筛选后的商品列表 + 结构化条件 `{price_max, color, rating_min}`

**实现**：LLM 解析 NL → 提取结构化条件 → 服务端按条件过滤列表 → 返回结果

**关键设计**：Prompt 需具备鲁棒性，能处理各种表达变体（如"不超过一千"、"黑色或深蓝"、"四分八以上"）

### 3.5 价格趋势模块 (Trend)

**输入**：商品 ID
**输出**：价格趋势分析 `{history_prices, avg_price, trend, suggestion, confidence}`

**Mock 历史数据**：为每个商品预生成 90 天历史价格数据（3 个平台各 30 天）

**AI 分析**：LLM 基于历史数据生成自然语言建议：
- "当前价格 ¥799，低于 90 天均价 ¥850（↓12%），**建议立即购买**"
- "当前价格 ¥899，高于 90 天均价 ¥820（↑9.6%），**预计 14 天内降价**，建议加入关注"

### 3.6 多目标智能识别模块 (Multi-Object Detection) ⭐ 差异化亮点

**输入**：用户上传的场景照片（如商场橱窗、街头穿搭、桌面摆拍）
**输出**：多个商品检测框 + 每个框的独立识别结果

**技术实现**（区域级检测策略）：
- 阶段 1：VLM 描述整张图内容
- 阶段 2：LLM 从描述中提取所有商品实体及其大致位置（左上/右上/中心等）
- 阶段 3：App 端根据位置信息渲染检测框动画
- 阶段 4：用户点击某框 → 独立调用识物模块获取详细属性

> 策略说明：纯 VLM 无法做精确像素级 bounding box。采用"区域级检测"（如"左上方白色运动鞋"）→ App 端用网格热区近似渲染 → 用户点击区域后进入单商品识别。既展示多目标理解能力，又避免需要真正的目标检测模型。

**Demo 震撼点**：拍一张图 → 满屏淡入检测框动画 → 点击任一框弹出详情卡片

### 3.7 AI 购物决策报告模块 (Decision Report) ⭐ 差异化亮点

**输入**：用户筛选后的最终商品列表 + 用户行为数据
**输出**：结构化决策报告（JSON）+ 精美分享卡片（PNG 图片）

**报告内容**：

```json
{
  "target_product": "Nike Air Max 黑色 42码",
  "best_choice": {
    "platform": "京东",
    "price": 749,
    "discount": "↓16%",
    "reason": "90天最低价，官方旗舰店"
  },
  "alternatives": [...],
  "ai_suggestion": "立即购买，未来14天涨价概率 73%",
  "confidence": 0.89,
  "generated_at": "2025-05-21T14:30:00Z"
}
```

**分享卡片生成**：
- 服务端将报告渲染为 HTML 模板 → 转 PNG 图片
- 支持一键保存到相册 + 分享到微信
- 卡片设计采用极简白底 + 品牌青点缀 + 清晰信息层级

### 3.8 多轮对话式 AI 导购模块 (AI Shopping Assistant) ⭐ 差异化亮点

**输入**：用户消息文本 + 当前会话上下文 + 当前商品状态
**输出**：AI 回复文本 + 可选的操作指令（如刷新列表、跳转页面）

**对话场景覆盖**：

| 用户输入 | AI 回复 | 操作 |
|---------|---------|------|
| "有别的颜色吗？" | "同款有白色、灰色、蓝色可选，白色当前最低价 ¥699" | 刷新列表为同色系列 |
| "和 Adidas Ultraboost 比怎么样？" | "Air Max 偏缓震，Ultraboost 偏回弹。你的使用场景是？" | 弹出对比卡片 |
| "预算只有 500" | "推荐降级款：特步动力巢，相似外观，当前价 ¥459" | 刷新列表为替代方案 |
| "这个价格值得买吗？" | "当前价低于 90 天均价 12%，且为官方旗舰店，建议购买" | 高亮最优选项 |

**上下文管理**：
- 使用 Redis 存储会话历史（TTL 24h）
- 每次对话携带最近 5 轮上下文 + 当前商品快照
- LLM Prompt 中注入"导购人格"：理性、简洁、以数据驱动建议

## 4. 数据模型与存储

### 4.1 数据库选型

| 存储 | 用途 | 理由 |
|------|------|------|
| **PostgreSQL** | 商品数据、用户行为、报告历史 | 结构化数据 + JSONB 灵活存储 AI 输出 |
| **Redis** | 会话缓存、图片临时存储、API 限流 | 对话上下文 TTL 24h、识物结果短期缓存 |

### 4.2 核心数据表

```sql
-- 商品表（Mock 数据 + 预留真实数据）
CREATE TABLE products (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    name VARCHAR(255) NOT NULL,
    brand VARCHAR(100),
    category VARCHAR(100),
    color VARCHAR(50),
    price DECIMAL(10,2),
    platform VARCHAR(50),  -- 'mock_taobao', 'mock_jd', 'real_taobao'...
    original_url TEXT,
    rating DECIMAL(2,1),
    tags JSONB,           -- ["自营", "官方", "包邮"]
    image_url TEXT,
    created_at TIMESTAMP DEFAULT NOW()
);

-- 历史价格表（用于趋势分析）
CREATE TABLE price_history (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    product_id UUID REFERENCES products(id),
    platform VARCHAR(50),
    price DECIMAL(10,2),
    recorded_at TIMESTAMP
);

-- 用户会话表（AI 导购上下文）
CREATE TABLE chat_sessions (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id VARCHAR(100),
    context JSONB,        -- 最近对话 + 当前商品快照
    expires_at TIMESTAMP
);

-- 决策报告表
CREATE TABLE decision_reports (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    session_id UUID,
    report_data JSONB,    -- 完整报告结构
    share_image_url TEXT,
    created_at TIMESTAMP DEFAULT NOW()
);
```

### 4.3 数据模拟策略

- **预设 200-300 条商品**：覆盖鞋服、数码、美妆、家居 4 大品类
- **每个商品 3 个平台价格**：淘宝/京东/拼多多，价格有合理差异
- **90 天历史价格**：用脚本批量生成，含波动、促销峰值、日常低谷
- **关键词匹配**：商品名 + 品牌 + 品类 + 颜色 做全文检索（PostgreSQL `tsvector`）
- **未知商品 Fallback**：LLM 动态生成参考价（标注"AI 参考价"）+ 推荐相似款

## 6. 错误处理策略

### 6.1 分层错误处理

| 层级 | 错误场景 | 处理策略 | 用户体验 |
|------|---------|---------|---------|
| **网络层** | 图片上传超时/失败 | 自动重试 2 次 → 提示"网络不稳，请重试" | 底部 Snackbar |
| **AI 层** | VLM 调用失败/超时 | 降级为"通用商品识别" → 只返回大类 | "AI 识别受限，已给出大致分类" |
| **AI 层** | LLM 结构化输出解析失败 | JSON Schema 校验失败 → 重试 1 次 → 仍失败则返回原始文本 | 展示原始描述，允许手动输入 |
| **数据层** | Mock 库无匹配商品 | 启动 Fallback 模式（LLM 生成参考价） | "AI 参考价" + 相似推荐 |
| **数据层** | 数据库连接失败 | 返回缓存数据 / 503 错误 | "服务暂时不可用" |
| **业务层** | 用户修正后仍无结果 | 扩大筛选条件 → 推荐"宽松匹配结果" | "未找到精确匹配，以下是相似款" |

### 6.2 优雅降级路径

```
理想路径：拍照 → 精确识别 → 精确比价 → 精确筛选
          ↓         ↓           ↓           ↓
降级路径：拍照 → 模糊识别 → 参考价生成 → 相似推荐
          ↓         ↓           ↓           ↓
兜底路径：拍照 → 仅大类识别 → 通用建议 → 热门推荐
```

无论哪条路径，App 都不会崩溃或空白，始终保持流畅体验。

### 6.3 识别纠错机制

| 机制 | 说明 |
|------|------|
| **自动置信度提示** | 识别结果旁显示"AI 置信度 87%"，低置信度（<70%）时主动提示用户核对 |
