## 二、核心链路端到端测试

### 2.1 拍照识物（单目标识别）

| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 1 | 首页点击「拍照识物」大按钮 | 进入相机拍摄界面 |
| 2 | 拍摄一张单商品照片（如怡宝矿泉水）| 显示 ScanLineOverlay 扫描线加载动画 |
| 3 | 等待识别完成 | 自动跳转 ResultScreen，显示商品名、品牌、品类、颜色 |
| 4 | 点击「查看比价」| 跳转 CompareScreen，显示多个平台的商品卡片 |
| 5 | 点击「价格走势」| 跳转 TrendScreen，显示折线图 + AI 分析 |
| 6 | 返回 ResultScreen，点击「AI 导购」| 跳转 ChatScreen，AI 能识别当前商品并导购 |

**通过标准**：全链路无报错，每个页面数据正常显示

---

### 2.2 多目标识别（6月2日重写功能）⭐重点测试

| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 1 | 首页点击「多目标识别」入口 | 进入相机拍摄界面 |
| 2 | 拍摄包含 **2-3 个商品** 的照片（如怡宝 + 资生堂护肤品并排放）| 显示 ScanLineOverlay 加载动画（扫描线 + 脉冲圆点）|
| 3 | 等待识别完成（约 8-10 秒）| 进入结果页： |
| | | ✅ 照片全屏显示，边缘有暗角效果（Vignette）|
| | | ✅ 每个商品位置有品牌青色气泡标签（序号 + 商品名）|
| | | ✅ 气泡有小三角箭头指向商品中心 |
| | | ✅ 气泡之间有连接线（如果间距 > 0）|
| | | ✅ 气泡依次弹入动画（Spring elasticOut），间隔约 120ms |
| 4 | 点击第一个气泡 | 跳转 ResultScreen，显示该商品信息 |
| 5 | 从 ResultScreen 返回 | 回到多目标识别结果页 |
| 6 | 点击第二个气泡 | 跳转该商品的 ResultScreen |

**边界测试**：
- [ ] 拍摄只有一个商品的照片 → 应直接跳转 ResultScreen（不经过多目标页）
- [ ] 拍摄没有商品的纯背景 → 应显示空结果（无气泡）
- [ ] 两个商品靠得很近 → 防重叠算法应自动向下偏移第二个气泡

**6/5 待优化项（暂不阻塞通过）**：
- 气泡样式进一步美化（安排在 6/5）
- 动画细节微调

### 2.3 AI 导购对话

| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 1 | 首页底部导航点击「聊天」| 进入 ChatScreen |
| 2 | 发送："帮我选一双运动鞋" | AI 回复，可能显示决策卡片 |
| 3 | 发送："对比一下 AF1 和 Dunk" | 显示橙色「对比分析」卡片 |
| 4 | 发送："AF1 什么时候买最划算" | 显示绿色「购买指南」卡片 |
| 5 | 连续对话 6 轮以上 | AI 能记住上下文，不重复提问 |
| 6 | 发送："生成报告" | 显示品牌青「AI决策报告」卡片，点击跳转 ReportScreen |

**通过标准**：AI 回复自然，决策卡片类型匹配用户意图，报告可正常跳转
