# 📦 Smart Price — 商品模拟数据说明

</div>

---

## 为什么使用模拟数据？

Smart Price 的多平台比价功能需要聚合京东、天猫、拼多多等平台的商品价格信息。由于以下原因，项目当前阶段采用**模拟数据**：

1. **缺乏合规 API**：主流电商平台（京东、天猫、拼多多）未提供公开免费的商品价格查询 API，企业级 API 接入需要商务合作和资质审核，周期较长
2. **禁止非法数据采集**：项目严格遵守技术合规边界，不编写任何形式的网络爬虫，不以违反网站服务条款的方式获取数据
3. **聚焦核心能力**：模拟数据使团队能将精力聚焦于 AI 识别、智能导购等核心创新功能，而非数据爬取

---

## 模拟数据格式

模拟数据采用结构化 JSON 格式，与真实电商平台的商品数据结构对齐，便于后续无缝切换至真实数据源：

```json
{
  "name": "Nike Air Force 1 '07",
  "price": 749,
  "original_price": 899,
  "platform": "京东",
  "tags": ["自营", "官方", "包邮"],
  "rating": 4.8,
  "sales": 100000,
  "url": "https://...",
  "image": "https://..."
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `name` | string | 商品名称 |
| `price` | float | 当前售价 |
| `original_price` | float | 原价（用于计算折扣）|
| `platform` | string | 电商平台：京东 / 天猫 / 拼多多 / 抖音电商 |
| `tags` | array | 标签：自营 / 官方 / 旗舰店 / 包邮 / 百亿补贴 |
| `rating` | float | 用户评分（1-5）|
| `sales` | int | 销量 |

---

## 数据生成逻辑

模拟数据通过 `MockDataProvider` 按以下规则生成：

1. **价格生成**：基于商品基准价，各平台随机浮动 ±10%~±20%，部分平台叠加"百亿补贴"折扣
2. **标签分配**：京东侧重"自营"，天猫侧重"旗舰店"，拼多多侧重"百亿补贴"
3. **评分生成**：4.5-5.0 之间正态分布，官方旗舰店评分略高
4. **销量生成**：基于品类热度，运动鞋 5万-50万，护肤品 1万-20万

---

## 切换至真实数据

项目采用 **Provider 模式** 封装数据源，切换至真实 API 时只需：

1. 实现 `BaseDataProvider` 接口（`get_products()`, `get_trend()`）
2. 在 `CompareService` 中替换 `MockDataProvider` 为新的 Provider
3. 保留原有的排序、筛选逻辑不变
