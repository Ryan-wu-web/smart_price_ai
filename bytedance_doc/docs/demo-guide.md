## 一、环境准备

### 1.1 启动后端

```bash
cd backend
source .venv/bin/activate  # Windows: .venv\Scripts\activate
uvicorn app.main:app --host 0.0.0.0 --port 8000
```

确认：访问 `http://localhost:8000/health` 返回 `{"status":"ok"}`

### 1.2 启动前端

```bash
cd android-app
flutter run
```

确认：App 成功安装到真机，首页正常显示

### 1.3 网络检查

确保手机和电脑在同一 WiFi，且 `constants.dart` 中的 `apiBaseUrl` 为电脑局域网 IP。

## 二、最小功能验证用例集

### 用例 1：拍照识物（单目标）⭐核心链路

**目的**：验证 AI 图像识别 + 属性提取 + 跨平台比价全链路

**操作路径**：

| 步骤 | 操作 | 预期结果 | 截图检查点 |
|------|------|---------|-----------|
| 1 | 首页点击「拍照识物」大按钮 | 进入相机拍摄界面 | 相机预览正常 |
| 2 | 拍摄一张单商品照片（如怡宝矿泉水）| 显示 ScanLineOverlay 扫描线加载动画 | 扫描线 + 脉冲圆点 + 阶段文字 |
| 3 | 等待识别完成 | 自动跳转 ResultScreen | 显示：商品名、品牌、品类、颜色 |
| 4 | 点击「查看比价」| 跳转 CompareScreen | 显示多平台商品卡片（价格、平台、评分）|
| 5 | 点击「价格走势」| 跳转 TrendScreen | 显示折线图 + AI 趋势分析 |
| 6 | 返回 ResultScreen，点击「AI 导购」| 跳转 ChatScreen | AI 能识别当前商品并导购 |

**通过标准**：全链路无报错，每个页面数据正常显示

### 用例 2：多目标识别 ⭐差异化亮点

**目的**：验证一张图识别多个商品 + 气泡标签可视化

**操作路径**：

| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 1 | 首页点击「多目标识别」入口 | 进入相机拍摄界面 |
| 2 | 拍摄包含 **2-3 个商品** 的照片 | 显示 ScanLineOverlay 加载动画 |
| 3 | 等待识别完成 | 进入结果页，显示：照片 + 暗角 + 气泡标签 + 连接线 |
| 4 | 观察气泡动画 | 气泡依次弹入（easeOutBack），锚点脉冲一次 |
| 5 | 点击第一个气泡 | 跳转该商品的 ResultScreen |
| 6 | 返回多目标页，点击第二个气泡 | 跳转第二个商品的 ResultScreen |

**通过标准**：气泡位置准确，动画流畅，点击跳转正常

**边界测试**：
- 只有一个商品 → 直接跳转 ResultScreen（不经过多目标页）
- 两个商品靠得很近 → 防重叠算法自动向下偏移

### 录制建议

- 使用真机录屏（MIUI/ColorOS 自带录屏功能）
- 保持画面稳定，操作节奏适中
- 关键动画（气泡弹入、流式聊天）停留 3-5 秒让评委看清
- 背景音乐可选，但不宜喧宾夺主

## 四、常见问题

**Q：识别超时怎么办？**
A：首次识别约 8-10 秒（VLM 调用），请确保 API Key 有效。同一商品二次拍照会命中缓存（< 0.1 秒）。

**Q：多目标识别气泡重叠？**
A：这是设计预期 — 防重叠算法会在密集场景自动向下偏移气泡。如仍不满意，可尝试让商品间距更大。

**Q：AI 聊天回复慢？**
A：SSE 流式响应已优化感知等待，实际等待时间与 LLM 推理速度相关（通常 5-12 秒）。
