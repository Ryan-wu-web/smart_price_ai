## 15 分钟问答应对策略

### 会的问题
- 先停顿 1 秒，组织语言
- 用"总-分-总"结构：先说结论，再解释原因，最后总结
- 适当引用项目中的具体代码或数据

### 不会的问题
- **不要装懂！**
- 标准话术："这个问题我目前了解不够深入，我的理解是...（简单说两句），后续我会重点研究这个方向。"
- 评委更看重你的诚实和学习态度

### 被质疑"AI 写的代码，你有什么贡献？"
- 标准话术："AI 确实帮我生成了大量代码，但项目的架构设计、技术选型、Prompt 工程、代码审查、测试验证都是我独立完成的。AI 是工具，我是决策者。"

## 紧急预案

| 情况 | 应对 |
|------|------|
| 后端启动失败 | 检查 `.env` API Key 是否过期；检查端口是否被占用 |
| 网络不通 | 提前下载源码到本地；用本地环境演示 |
| 真机连不上 | 用 Android 模拟器代替；或者录屏演示 |
| 评委问超纲问题 | 诚实承认 + 表达学习意愿 |
| 紧张忘词 | 深呼吸，看架构图回忆；可以说"让我想一下" |
