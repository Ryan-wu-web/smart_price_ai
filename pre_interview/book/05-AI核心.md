# Day 5：AI 核心 — 项目的大脑

> 目标：理解 VLM 和 LLM 的区别，Prompt 工程的基本原理，以及项目中 AI 是怎么识别商品和对话的。

---

## 1. 大模型两种类型

| 类型 | 全称 | 能力 | 项目中用途 |
|------|------|------|-----------|
| **VLM** | Visual Language Model（视觉语言模型）| 看懂图片 + 理解文字 | 识别商品照片中的品牌、颜色、款式 |
| **LLM** | Large Language Model（大语言模型）| 理解文字 + 生成文字 | 和用户对话，生成购买建议 |

**一句话区别**：VLM 有"眼睛"能看图，LLM 只有"嘴巴"会说话。

---

## 2. 大模型是怎么工作的？（简化版）

**核心原理**：Transformer 架构。

**不用懂数学，记住三个概念**：

1. **Token**：模型处理的最小单位。一个汉字 ≈ 1 个 token。
2. **注意力机制**：模型看一句话时，会重点关注关键词。
3. **生成过程**：模型每次预测下一个最可能的 token，一个一个拼成回答。

**为什么 AI 回复要时间？**
- 生成 100 个 token，就要预测 100 次
- 每次预测都要做大量计算
- 所以聊天要等 5-10 秒

## 3. Prompt 工程 — 跟 AI "说话的艺术"

Prompt = **你给 AI 的指令**。同样的模型，不同的 Prompt，输出质量天差地别。

**好 Prompt 的四个要素**：

| 要素 | 说明 | 项目中的例子 |
|------|------|------------|
| **角色设定** | 告诉 AI 它是谁 | "你是一位专业的商品识别专家" |
| **任务描述** | 告诉 AI 要干什么 | "请根据图片描述提取结构化信息" |
| **输出格式** | 告诉 AI 怎么输出 | "以 JSON 格式输出" |
| **约束条件** | 告诉 AI 不能干什么 | "如果无法识别品牌，brand 设为空字符串，不要猜测" |

**项目中识别 Prompt 的核心**：
```
角色：商品识别专家
任务：提取品牌、品类、颜色、款式
格式：JSON
约束：brand 不能猜；category 要具体；只输出 JSON 不要解释
```

---

## 4. 为什么模型输出不稳定？

**问题**：同样的图片，有时识别出 Nike，有时识别出 Adidas？

**原因**：
- 大模型本质是"概率预测"，每次预测有随机性
- 温度参数（temperature）越高，回答越随机

**项目中怎么解决？**
1. **降低 temperature**：让模型更"保守"
2. **Few-shot 示例**：给模型看几个正确例子
3. **JSON Schema 约束**：强制输出 JSON
4. **容错回退**：JSON 解析失败时，用正则提取关键字段

## 5. SSE 流式输出原理

**普通调用**：等 10 秒，然后一次性看到全部回答。
**SSE 流式**：立刻看到第一个字，然后逐字出现。

**实现方式**：
- 后端：调用模型时设置 `stream=true`，模型每生成一个 token 就推送一次
- 前端：建立 SSE 连接，收到一个字就更新页面

**答辩话术**：
> "为了降低用户感知等待时间，我们采用 SSE 流式传输。后端通过 FastAPI StreamingResponse 逐字推送模型输出，前端实时接收并追加渲染，实现类 ChatGPT 的打字机效果。"
