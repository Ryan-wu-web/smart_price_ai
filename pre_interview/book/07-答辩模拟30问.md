# Day 7：答辩模拟 — 30个必问问题

> 目标：覆盖评委最可能问的问题，每个问题都有"一句话版本"和"详细版本"。

---

## 一、项目理解类（必问）

### Q1：用一句话介绍你的项目
**一句话**：Smart Price 是一款 AI 拍照识物购物助手，帮用户拍照识别商品、比价、AI 导购。

### Q2：项目解决什么痛点？
**答案**：线下看到商品不知道值不值得买、网上搜同款费时费力、不知道怎么选最划算。

### Q3：项目有哪些核心功能？
**答案**：拍照识物、多目标识别、跨平台比价、AI 导购对话、价格走势、决策报告。

### Q4：你和现有 App（淘宝拍照、京东识物）有什么区别？
**答案**：
- 他们只有单目标识别，我们支持多目标
- 他们有比价但没有 AI 导购，我们有对话式导购
- 我们支持自然语言追加筛选，交互更自然

## 二、技术选型类（高频）

### Q5：为什么前端选 Flutter？
**答案**：跨平台一套代码、CustomPainter 自绘能力强（气泡标签）、热重载开发效率高。

### Q6：为什么后端选 FastAPI？
**答案**：异步原生（高并发 SSE）、自动文档（Swagger）、Pydantic 类型安全。

### Q7：为什么 AI 用火山引擎 Doubao？
**答案**：VLM+LLM 统一、中文优化好、原生支持 SSE 流式。

### Q8：为什么用 SQLite 而不是 MySQL/PostgreSQL？
**答案**：项目规模小，SQLite 零配置、轻量、足够用。后期可无缝迁移。

---

## 三、架构设计类（高频）

### Q9：系统怎么分层的？
**答案**：客户端层（Flutter UI）→ 服务端层（FastAPI 业务）→ AI 引擎层（Doubao）→ 数据层（SQLite+Mock）。

### Q10：前后端怎么通信？
**答案**：HTTP RESTful API，JSON 格式。聊天用 SSE 流式传输。

### Q11：如果用户量变大 10 倍，系统怎么扩展？
**答案**：
- 前端：Flutter 本身无状态，加 CDN 加速静态资源
- 后端：FastAPI 用 Gunicorn + Uvicorn 多进程部署
- 数据库：SQLite 换 PostgreSQL + Redis 缓存
- AI：加请求队列，限流保护

### Q12：你的模块是怎么划分的？
**答案**：Router 收请求 → Service 处理业务 → Client 调外部 API → 数据库存数据。关注点分离，便于测试和替换。

## 四、AI 技术类（重点）

### Q13：VLM 和 LLM 有什么区别？
**答案**：VLM 能看懂图片（视觉+语言），LLM 只能处理文字（纯语言）。我们识别商品用 VLM，对话用 LLM。

### Q14：Prompt 怎么设计的？
**答案**：四要素——角色设定（商品识别专家）、任务描述（提取属性）、输出格式（JSON）、约束条件（brand 不能猜）。

### Q15：模型输出不稳定怎么办？
**答案**：降低 temperature、给 Few-shot 示例、JSON Schema 约束、JSON 解析失败时正则回退。

### Q16：SSE 流式怎么实现的？
**答案**：后端 FastAPI StreamingResponse 逐字 yield，前端 HTTP SSE 逐字接收渲染。

### Q17：多目标识别怎么实现的？
**答案**：VLM 返回多个商品的属性+坐标，前端布局算法自动定位气泡，自动避碰。

## 五、工程细节类（深挖）

### Q18：缓存怎么做的？
**答案**：dHash 感知哈希做 key，SQLite 存结果。容忍重新拍照的差异，二次识别 < 0.1 秒。

### Q19：dHash 和 MD5 有什么区别？
**答案**：MD5 对任何像素变化都敏感，重新拍照角度一变就完全不同。dHash 比较相邻像素亮度差异，容忍角度/亮度变化。

### Q20：图片怎么压缩的？
**答案**：缩到 600px 宽，JPEG 质量 75%，转 RGB。从 3MB 降到 ~200KB。

### Q21：连接池复用怎么实现的？
**答案**：`httpx.AsyncClient` 实例级持有，不每次新建，省 TCP 握手时间。

### Q22：气泡标签的布局算法怎么做的？
**答案**：初始放锚点上方 → 上方不够翻转到下方 → 水平重叠向下推移 → 边缘越界水平吸附。

## 六、个人贡献类（必问）

### Q23：这个项目你做了什么？
**答案**：前后端全部开发、UI 设计、AI Prompt 设计、测试验证。AI 辅助编码，但架构设计、代码审查、调试优化都是我自己完成的。

### Q24：AI 写了多少代码？你写了多少？
**答案**：AI 生成约 80% 的代码，但我负责架构设计、Prompt 工程、代码审查、调试优化、测试验证。关键决策（技术选型、模块划分、缓存策略）都是我做的。

### Q25：你怎么保证代码质量？
**答案**：
- 设计阶段：先写文档再写代码（Superpowers 方法论）
- 开发阶段：每步 flutter analyze / pytest，0 issues 才提交
- 测试阶段：两轮全量真机测试，记录 Bug 并修复

### Q26：遇到的最大困难是什么？怎么解决的？
**答案**：多目标识别的布局算法。气泡容易重叠。解决：先排序再逐层检查碰撞，上方空间不足自动翻转，边缘自动吸附。反复真机调试验证。

## 七、开放思考题（加分）

### Q27：如果让你继续做，下一步优化什么？
**答案**：
- 接入真实电商平台 API（替代 Mock 数据）
- 增加语音输入（用户直接说话识别）
- 用户行为分析（推荐更精准）

### Q28：你的项目有什么技术亮点？
**答案**：
- 多目标识别 + 气泡标签可视化（差异化功能）
- SSE 流式 AI 聊天（极致体验）
- dHash 感知哈希缓存（工程优化）
- 完整的 AI Pipeline 设计（任务编排）

### Q29：AI Coding 给你带来了什么？
**答案**：效率提升 3-5 倍，但更需要把控质量。我建立了 Superpowers 流程：先设计再编码、频繁审查、逐步验证。

### Q30：如果评委让你现场改一个功能，你怎么办？
**答案**：先理解需求 → 定位相关模块 → 小步修改 → 本地验证 → 提交。如果时间紧，优先改前端（Flutter 热重载快）。
